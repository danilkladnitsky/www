# Почему TypeScript того стоит

Даже если поначалу кажется, что типы — лишняя возня, они быстро начинают окупаться.

## Меньше багов на этапе разработки

IDE подсказывает поля объектов и аргументы функций. Опечатки и неверные типы ловятся до запуска, а не в рантайме у пользователя.

## Удобный рефакторинг

Переименование типа или интерфейса обновляется по всему проекту. Меняешь контракт API — компилятор покажет все места, которые нужно поправить.

## Живая документация

Типы и интерфейсы описывают форму данных. Новый разработчик (или ты через полгода) смотрит на сигнатуры и понимает, что куда передавать, без лишнего поиска по коду.

Не обязательно типизировать всё на 100%. Начни с новых модулей и публичных API — уже это даст заметную пользу.
