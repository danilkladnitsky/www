# Простой Git-воркфлоу для команд

Не обязательно внедрять сложные схемы. Достаточно договориться о нескольких вещах.

## Ветки

- **main** (или **master**) — всегда деплоится, в неё только через merge/pull request.
- Ветка под задачу: `feature/краткое-название` или `fix/что-починили`. Одна ветка — одна логическая задача.

## Коммиты

- Сообщение в повелительном наклонении: «Добавить валидацию формы», а не «добавил валидацию».
- Маленькие коммиты: проще откатить, проще ревьюить. Один коммит — один осмысленный шаг.

## Перед пушем

Сделай `git pull --rebase` (или обычный pull, если так принято), прогнать линтер и тесты. Так в общую историю попадает уже проверенный код.

Такой подход не требует специальных инструментов и легко объяснить новым участникам команды.
